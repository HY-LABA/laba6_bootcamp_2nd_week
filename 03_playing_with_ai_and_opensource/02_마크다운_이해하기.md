# 📄 마크다운(.md)이란?

## 정의

**마크다운(Markdown)** 은 `#`, `-`, `**`, `>` 같은 **간단한 기호만으로**
글의 구조(제목·목록·강조·인용 등)를 표현하는 경량 문서 형식입니다.

- 확장자는 `.md`
- 복잡한 프로그램 없이 **메모장으로도** 쓸 수 있음
- GitHub, 블로그, 문서 사이트 어디서나 예쁘게 렌더링됨

| 이렇게 쓰면          | 이렇게 보입니다      |
| -------------------- | -------------------- |
| `# 제목`             | 큰 제목              |
| `- 항목`             | • 항목 (목록)        |
| `**중요**`           | **중요** (굵게)      |
| `> 인용`             | 인용 블록            |
| `[링크](주소)`       | 클릭 가능한 링크     |

---

## 왜 AI 시대에 마크다운이 중요할까?

AI(Claude, Codex, Copilot 등)는 **마크다운 구조를 아주 잘 이해합니다.**
제목·목록·강조로 정리된 문서를 주면, 사람의 의도를 훨씬 정확히 파악하죠.

그래서 요즘 AI 도구들은 `README.md`, `CLAUDE.md`, `AGENT.md` 같은
**마크다운 파일을 규칙서처럼 읽고 그대로 따릅니다.**

---

## 🧪 데모: "AI가 이 파일을 읽으니 대답이 달라졌다"

이걸 눈으로 보여주기 가장 좋은 예시는 **바로 이 저장소의 `AGENT.md`** 입니다.

### 방법

1. 루트의 [`AGENT.md`](../AGENT.md) 에는 이런 규칙이 적혀 있습니다:
   > "AI는 항상 **'LABA 6기 친구들 안녕! 내가 대답을 할게!'** 로 답변을 시작한다."
2. `CLAUDE.md` 안의 `@AGENT.md` 한 줄이 이 규칙을 AI에게 자동으로 읽힙니다.

### 결과 비교

| 구분 | AI의 대답 |
| ---- | --------- |
| **파일을 안 읽었을 때** | "안녕하세요! 무엇을 도와드릴까요?" (평범한 답) |
| **AGENT.md를 읽은 뒤** | **"LABA 6기 친구들 안녕! 내가 대답을 할게!"** 로 시작 ✅ |

똑같은 질문을 해도 답의 첫 문장이 **눈에 띄게 바뀝니다.**
→ "마크다운 파일 하나가 AI의 행동을 실제로 바꾼다"는 걸 바로 확인할 수 있어요.

### 다른 데모 아이디어 (원한다면)

- 파일에 **"모든 답변 끝에 🐢 이모지를 붙여라"** 라고 적어두기
- **"코드에는 항상 한글 주석을 달아라"** 라고 적어두기
- 우리 팀만의 용어집(예: "배포 = deploy") 을 적어두고 AI가 그 단어를 쓰는지 보기

> 규칙을 적기 **전/후**로 같은 질문을 던져 답을 비교하면,
> 마크다운 문서가 AI에게 얼마나 강력한 '지시서'인지 확실히 체감됩니다.
