# SO-ARM + LeRobot 실습해보기

> 텔레오퍼레이션으로 데이터 수집 → HuggingFace 업로드 → DGX에서 ACT 학습 → DGX에서 추론.
> **lerobot 이 이미 CLI 도구(`lerobot-*`)를 제공**하므로, 아래처럼 명령어로 각 단계를 실행합니다.

## 💡 lerobot 이 제공하는 CLI 도구들

lerobot 을 설치하면 아래 명령어들이 바로 생깁니다 (별도 스크립트 작성 불필요):

| 명령어 | 역할 |
|---|---|
| `lerobot-find-port` | 팔이 연결된 USB 포트(`/dev/ttyACM*`) 찾기 |
| `lerobot-calibrate` | 팔 캘리브레이션 |
| `lerobot-teleoperate` | 리더로 팔로워 조종 (셋업 검증) |
| `lerobot-record` | 텔레옵 데이터 수집 + HF 업로드 |
| `lerobot-train` | 정책(ACT/SmolVLA 등) 학습 |

## 0. DGX 접속 + 환경 활성화

```bash
ssh laba@172.16.142.85
source ~/smolvla/dgx/.arm_finetune/bin/activate
```

- 활성화되면 프롬프트에 `(.arm_finetune)` 이 붙습니다.
- 이 팔은 캘리브레이션이 이미 저장돼 있어(`--robot.id`/`--teleop.id` 로 자동 로드) 재캘리브레이션 불필요.

## 1. 포트·카메라 지정 (세션마다 한 번) ★중요★

`/dev/ttyACM0` 같은 번호는 재부팅·재연결 때 뒤바뀔 수 있습니다.
그래서 **시리얼에 묶여 안 바뀌는 `by-id` 경로**를 셸 변수로 한 번만 정의해두고, 이후 모든 명령에서 재사용합니다.

```bash
# (1) 연결된 팔의 by-id 경로 확인
ls -l /dev/serial/by-id/
# (2) 위 결과를 보고 follower / leader 를 변수로 지정 (세션마다 1회)
FOLLOWER=/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6056701-if00
LEADER=/dev/serial/by-id/usb-1a86_USB_Single_Serial_5B42138566-if00

# (3) 카메라 인덱스 확인 후 카메라 설정도 변수로
v4l2-ctl --list-devices
CAMS="{ top: {type: opencv, index_or_path: 2, width: 480, height: 640, fps: 30, color_mode: rgb, fourcc: MJPG, rotation: -90}, wrist: {type: opencv, index_or_path: 0, width: 640, height: 480, fps: 30, color_mode: rgb, fourcc: MJPG, rotation: 180}}"
```

## 2. 텔레옵으로 셋업 검증 (데이터 저장 X) ★필수★

```bash
lerobot-teleoperate \
  --robot.type=so101_follower --robot.port=$FOLLOWER --robot.id=leftarm_test_follower \
  --robot.cameras="$CAMS" \
  --teleop.type=so101_leader --teleop.port=$LEADER --teleop.id=leftarm_test_leader \
  --display_data=false
```

→ 리더를 움직였을 때 팔로워가 정상 추종하면 OK. (실패 시 팔 전원·케이블·포트부터 확인)

**카메라 top/wrist 확인까지 하려면**

```bash
lerobot-teleoperate \
  --robot.type=so101_follower --robot.port=$FOLLOWER --robot.id=leftarm_test_follower \
  --robot.cameras="$CAMS" \
  --teleop.type=so101_leader --teleop.port=$LEADER --teleop.id=leftarm_test_leader \
  --display_data=true
```

## 3. 데이터 수집 + HF 업로드

```bash
lerobot-record \
  --robot.type=so101_follower --robot.port=$FOLLOWER --robot.id=leftarm_test_follower \
  --robot.cameras="$CAMS" \
  --teleop.type=so101_leader --teleop.port=$LEADER --teleop.id=leftarm_test_leader \
  --dataset.repo_id=BaboGaeguri/laba6_bootcamp_2nd_week \
  --dataset.single_task="Pick up the object and place it" \
  --dataset.num_episodes=10 \
  --dataset.fps=30 \
  --dataset.episode_time_s=60 \
  --dataset.reset_time_s=5 \
  --dataset.push_to_hub=true \
  --display_data=false
```

- `--dataset.push_to_hub=true` → 수집이 끝나면 `BaboGaeguri/laba6_bootcamp_2nd_week` 로 자동 업로드.
- 수집 중 조작: 에피소드 성공 → `→`(다음) / 실패 → `←`(재수집) / `ESC` 종료.
- ⚠️ **이 키보드 조작(→/←/ESC)은 DGX 모니터가 연결된 화면에서 실행해야 작동합니다.** 순수 SSH(헤드리스)로 돌리면 `pynput`이 화면(`DISPLAY`)이 없어 키 입력을 못 받고, 에피소드가 타이머(`episode_time_s`)로만 자동으로 넘어갑니다. **수동 제어하려면 DGX 앞 터미널에서** 실행하세요.
- 녹화 끝에 Hub 업로드가 진행되니, 그때 `Ctrl+C`로 끊지 마세요 (업로드 중단됨).

### 이어서 더 수집하기 (resume)

이미 만든 데이터셋에 **에피소드를 추가**하려면, 위 명령에 **`--resume=true`** 만 붙입니다.

```bash
lerobot-record \
  --robot.type=so101_follower --robot.port=$FOLLOWER --robot.id=leftarm_test_follower \
  --robot.cameras="$CAMS" \
  --teleop.type=so101_leader --teleop.port=$LEADER --teleop.id=leftarm_test_leader \
  --dataset.repo_id=BaboGaeguri/laba6_bootcamp_2nd_week \
  --dataset.single_task="Pick up the object and place it" \
  --dataset.num_episodes=10 \
  --dataset.fps=30 \
  --dataset.episode_time_s=60 \
  --dataset.reset_time_s=5 \
  --dataset.push_to_hub=true \
  --resume=true \
  --display_data=false
```

- 예: 처음 10개 → 이 명령으로 10개 더 = 총 20개. 원하는 개수만큼 반복 실행 가능.
- **카메라·task·fps 등은 첫 수집과 동일하게** 유지해야 데이터셋이 일관됩니다 (특히 `$CAMS` 회전값).
- 로컬 데이터셋 캐시(`$HF_HOME/lerobot/BaboGaeguri/laba6_bootcamp_2nd_week`)를 읽어 이어붙이고, 다시 Hub로 push 합니다.

## 4. ACT 학습 (DGX)

포트가 필요 없는 단계라 변수도 필요 없습니다.

```bash
lerobot-train \
  --policy.type=act \
  --dataset.repo_id=BaboGaeguri/laba6_bootcamp_2nd_week \
  --dataset.video_backend=pyav \
  --policy.repo_id=BaboGaeguri/laba6_bootcamp_2nd_week_act \
  --batch_size=8 \
  --steps=30000 \
  --num_workers=8 \
  --output_dir=$HOME/smolvla/dgx/outputs/laba6_bootcamp_2nd_week_act \
  --job_name=laba6_bootcamp_2nd_week_act \
  --policy.device=cuda \
  --wandb.enable=true
```

- ACT 는 사전학습 가중치 없이 처음부터 학습 (`--policy.path` 불필요).
- **`--policy.repo_id` 는 학습된 모델을 올릴 HF Hub 위치** — 이게 없으면 `'policy.repo_id' argument missing` 에러가 납니다. (Hub에 안 올리려면 대신 `--policy.push_to_hub=false`)
- **`--dataset.video_backend=pyav`** — 이 DGX의 torchcodec(기본 디코더)가 최신 torch 2.10과 비호환(`Could not load libtorchcodec` / `undefined symbol`)이라, 비디오 디코딩을 **PyAV로 우회**합니다. 이거 없으면 데이터 로딩 단계에서 크래시.
- ⚠️ **`--output_dir`/`--policy.path` 는 `~` 대신 `$HOME` 사용** — 셸은 `--opt=~/경로` 처럼 `=` 뒤의 `~` 를 확장하지 않아, `~` 를 쓰면 `/home/laba/~/...` 라는 literal `~` 폴더가 생겨 `FileExistsError` 등 엉뚱한 문제가 납니다. `$HOME` 은 변수라 항상 확장됩니다.
- 체크포인트: `$HOME/smolvla/dgx/outputs/laba6_bootcamp_2nd_week_act/checkpoints/`
- 데이터가 적으면 `--steps` 를 줄이세요 (예: 4~10개면 `5000` 이면 충분·빠름). 많으면 2~3만.
- 🔁 **재실행 시**: 출력 폴더가 이미 있으면 `FileExistsError` 가 납니다. `rm -rf $HOME/smolvla/dgx/outputs/laba6_bootcamp_2nd_week_act` 로 지우거나 `--output_dir`/`--job_name` 을 새 이름으로 바꾸세요.

## 5. 추론 (DGX, 학습된 정책으로 실제 팔 구동)

```bash
lerobot-record \
  --robot.type=so101_follower --robot.port=$FOLLOWER --robot.id=leftarm_test_follower \
  --robot.cameras="$CAMS" \
  --dataset.repo_id=BaboGaeguri/laba6_bootcamp_2nd_week_eval \
  --dataset.single_task="Pick up the object and place it" \
  --dataset.num_episodes=3 \
  --dataset.push_to_hub=false \
  --policy.path=$HOME/smolvla/dgx/outputs/laba6_bootcamp_2nd_week_act/checkpoints/last/pretrained_model \
  --display_data=false
```

- 추론은 **리더 없이 팔로워만** 연결해도 됩니다 (정책이 리더 역할) → `--teleop.*` 생략.
- 카메라 인자(`$CAMS`)는 **학습 때와 동일**해야 합니다.
