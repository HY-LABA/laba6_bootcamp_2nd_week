# 📂 파일 위치(경로)와 자주 쓰는 bash 명령어

## 1. 파일 위치 = "경로(Path)"

리눅스의 모든 파일/폴더는 **`/` (루트)** 에서 시작하는 주소를 가집니다.

```
/home/laba/Desktop/project/main.py
└┬┘ └─┬─┘ └──┬──┘ └──┬──┘ └──┬──┘
 루트  홈   사용자   폴더    파일
```

### 꼭 알아둘 기호

| 기호 | 의미 | 예시 |
| ---- | ---- | ---- |
| `/` | 최상위(루트) 폴더 | `/home`, `/dev` |
| `~` | 내 홈 폴더 (`/home/사용자이름`) | `~/Desktop` = `/home/laba/Desktop` |
| `.` | **현재 위치**(지금 있는 폴더) | `./main.py` = 지금 폴더의 main.py |
| `..` | **한 단계 위** 폴더(부모) | `cd ..` = 상위 폴더로 이동 |

### 절대경로 vs 상대경로

- **절대경로**: `/` 부터 다 적음 → `/home/laba/Desktop/main.py` (어디서든 똑같음)
- **상대경로**: 현재 위치 기준 → `./main.py`, `../images/logo.png`

> 💡 지금 내 위치가 궁금하면 `pwd` (print working directory), 위치 이동은 `cd`.

## 2. 자주 쓰는 bash 명령어

### 📁 이동 · 탐색

| 명령어 | 하는 일 | 예시 |
| ---- | ---- | ---- |
| `cd` | 폴더 이동 (change directory) | `cd ~/Desktop` / `cd ..` (상위로) / `cd` (홈으로) |
| `ls` | 현재 폴더의 파일·폴더 목록 보기 | `ls` / `ls -l` (자세히) / `ls -a` (숨김파일까지) |

### 🗂️ 만들기 · 지우기

| 명령어 | 하는 일 | 예시 |
| ---- | ---- | ---- |
| `mkdir` | **새 폴더** 만들기 (make directory) | `mkdir project` / `mkdir -p a/b/c` (중간 폴더까지 한 번에) |
| `touch` | **새 (빈) 파일** 만들기 | `touch main.py` / `touch a.txt b.txt` (여러 개) |
| `rm` | 파일·폴더 **지우기** (remove) | `rm 파일.txt` / `rm -r 폴더` (폴더째) / `rm *.log` (여러 개) |

> ⚠️ **`rm` 은 휴지통을 거치지 않고 바로 영구 삭제**됩니다. 특히 `rm -r` 은 폴더 전체를 지우니 경로를 꼭 확인하세요.
> `rm -rf /` 같은 명령은 시스템을 통째로 날릴 수 있어 **절대 금지**입니다.

### 🔎 찾기 · 출력

| 명령어 | 하는 일 | 예시 |
| ---- | ---- | ---- |
| `grep` | 텍스트에서 특정 단어 찾기 | `grep "error" log.txt` / `ls \| grep .py` |
| `echo` | 글자를 화면에 출력 / 변수 확인 | `echo 안녕` / `echo $PATH` (환경변수 값 보기) |

### 🔐 권한 · 시스템

| 명령어 | 하는 일 | 예시 |
| ---- | ---- | ---- |
| `sudo` | **관리자 권한**으로 실행 (superuser do) | `sudo apt install git` (설치 등 권한 필요할 때) |
| `shutdown` | 시스템 종료 / 재부팅 | `sudo shutdown now` (즉시 종료) / `sudo reboot` (재부팅) |

> ⚠️ `sudo` 는 시스템을 바꿀 수 있어 강력합니다. 비밀번호를 물어보며, 남의 명령을 함부로 `sudo` 로 실행하지 마세요.

### 🐍 개발 · 실행

| 명령어 | 하는 일 | 예시 |
| ---- | ---- | ---- |
| `source` | 스크립트/설정을 **현재 셸에 불러오기** | `source venv/bin/activate` (가상환경 켜기) / `source ~/.bashrc` |
| `pip` | 파이썬 패키지 설치·관리 | `pip install python-dotenv` / `pip list` |
| `code` | **VSCode 열기** | `code .` (현재 폴더를 VSCode로 열기) / `code main.py` |
| `claude` | 터미널에서 **Claude Code** 실행 | `claude` (대화 시작) |
| `git` | 버전 관리 (코드 이력 저장·공유) | `git status` / `git add .` / `git commit -m "메시지"` |

### 🤖 로봇 · 장치 (로봇 프로젝트에서 자주 등장)

- **`/dev/ttyACM*`** — 명령어가 아니라 **장치 파일(경로)** 입니다.
  USB로 연결된 시리얼 장치(아두이노, 마이크로컨트롤러, 로봇 보드 등)가 여기에 잡힙니다.
  ```bash
  ls /dev/ttyACM*     # 연결된 장치 확인 (ttyACM0, ttyACM1 ...)
  ls /dev/ttyUSB*     # USB-시리얼 어댑터는 ttyUSB* 로 잡히기도 함
  ```
  - 리눅스는 "**모든 것을 파일로**" 다룹니다. 그래서 로봇 하드웨어도 `/dev/...` 파일로 접근해요.
  - 권한 문제로 안 열리면 사용자에게 권한을 주거나(`dialout` 그룹) 임시로 `sudo` 를 씁니다.

## 3. 알아두면 편한 팁

- `Tab` 키 → 파일/폴더 이름 **자동완성**
- `↑` / `↓` 방향키 → 이전에 쓴 명령어 다시 불러오기
- `명령어 --help` 또는 `man 명령어` → 사용법 보기 (예: `ls --help`)
- `Ctrl + C` → 실행 중인 명령 강제 중단
- `clear` → 터미널 화면 깨끗이 비우기
