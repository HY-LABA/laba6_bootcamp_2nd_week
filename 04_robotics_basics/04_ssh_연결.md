# 🔌 SSH 연결

## SSH란?

**SSH(Secure Shell)** = 다른 컴퓨터에 **네트워크로 원격 접속해서, 그 컴퓨터의 터미널을
내 앞에서 쓰는 것처럼** 조작하는 기술입니다. (통신은 암호화되어 안전함)

- 로봇/서버는 보통 **모니터·키보드가 없습니다.** → SSH로 내 노트북에서 접속해 제어
- 로봇이 옆방·먼 곳에 있어도, 같은 네트워크(또는 인터넷)면 접속 가능

```
[내 노트북] ──── SSH(암호화) ────▶ [로봇/서버 우분투]
   여기서 명령 입력                 저기서 실제 실행
```

## 접속하기 전에 필요한 것

1. 접속 **대상 컴퓨터의 정보**
   - 사용자 이름 (예: `laba`)
   - IP 주소 (예: `192.168.0.15`) — 대상에서 `ip addr` 또는 `hostname -I` 로 확인
2. 대상 컴퓨터에 **SSH 서버가 켜져 있어야** 함
   ```bash
   # (대상 컴퓨터에서 1회 설치·실행)
   sudo apt install openssh-server
   sudo systemctl enable --now ssh
   ```

## 접속 방법

내 컴퓨터 터미널에서:

```bash
ssh 사용자이름@IP주소
# 예시
ssh laba@192.168.0.15
```

- 처음 접속 시 `yes` 입력 → 비밀번호 입력 → 접속 완료!
- 접속되면 프롬프트가 **대상 컴퓨터 이름으로 바뀝니다.** (이제 입력하는 명령은 원격에서 실행됨)
- 포트를 바꿨다면: `ssh -p 2222 laba@192.168.0.15`
- 접속 끊기: `exit` 입력

## 자주 쓰는 관련 기능

| 상황 | 명령어 |
| ---- | ---- |
| 파일을 원격으로 복사 (보내기) | `scp 파일 laba@192.168.0.15:~/Desktop/` |
| 파일을 원격에서 가져오기 | `scp laba@192.168.0.15:~/log.txt .` |
| 매번 비밀번호 입력이 귀찮을 때 | **SSH 키** 등록 (`ssh-keygen` → `ssh-copy-id laba@192.168.0.15`) |

## 💡 VSCode로 원격 접속하기

명령어가 어렵다면, VSCode 확장 **`Remote - SSH`** 를 쓰면
**원격 로봇의 폴더를 내 VSCode 창에서 그대로 열어** 편집·실행할 수 있습니다.

1. VSCode에서 `Remote - SSH` 확장 설치
2. `Ctrl+Shift+P` → **Remote-SSH: Connect to Host** → `laba@192.168.0.15` 입력
3. 접속되면 원격 컴퓨터의 파일을 GUI로 편집 (터미널 + 파일 탐색 모두 사용 가능)

> 정리: **SSH = 원격 컴퓨터의 터미널을 빌려 쓰는 것.**
> 모니터 없는 로봇을 다룰 때 필수 기술입니다.
