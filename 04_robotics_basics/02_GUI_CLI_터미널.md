# 🖥️ GUI / CLI 그리고 터미널

## GUI vs CLI

컴퓨터에게 명령을 내리는 두 가지 방식입니다.

| 구분 | GUI (Graphical User Interface) | CLI (Command Line Interface) |
| ---- | ---- | ---- |
| 방식 | **마우스로 클릭** (아이콘, 버튼, 창) | **키보드로 명령어 입력** (글자) |
| 예시 | 폴더 아이콘 더블클릭해서 열기 | `cd 폴더이름` 입력해서 이동 |
| 장점 | 직관적, 배우기 쉬움 | 빠르고, 자동화·원격제어에 강함 |
| 단점 | 반복 작업·원격제어에 불리 | 명령어를 배워야 함 |

> 둘 다 "컴퓨터에 일을 시키는 것"은 같습니다. 방법이 마우스냐 글자냐의 차이일 뿐이에요.

## 왜 개발자는 CLI를 쓸까?

- **빠름** — 창을 여러 번 클릭할 일을 명령어 한 줄로 끝냄
- **자동화** — 여러 명령을 스크립트로 묶어 반복 실행 가능
- **원격제어** — 모니터 없는 로봇/서버에 SSH로 접속해 글자만으로 제어 (→ [`04_ssh_연결.md`](04_ssh_연결.md))
- **정확함** — "어느 폴더에서 무엇을 했는지"가 명령어로 명확히 남음

## 터미널이란?

**터미널(Terminal)** = CLI로 **명령어를 입력하는 창**입니다.

- 우분투에서 열기: `Ctrl + Alt + T`
- VSCode 안에서 열기: 상단 메뉴 `Terminal → New Terminal` (또는 `` Ctrl + ` ``)
- 프롬프트 예시:
  ```
  babogaeguri@babogaeguri-950QED:~/Desktop$
  ```
  | `babogaeguri` | `@` | `babogaeguri-950QED` | `:` | `~/Desktop` | `$` |
  | 사용자 이름 | | 컴퓨터(호스트) 이름 | | **현재 위치**(경로) | 명령 입력 대기 |

  → `$` 뒤에 명령어를 입력하고 `Enter` 를 누르면 실행됩니다.

> 참고: **셸(Shell)** 은 입력한 명령어를 해석해 실행해주는 프로그램(우분투 기본은 **bash**),
> **터미널**은 그 셸과 대화하는 창이라고 이해하면 됩니다.
